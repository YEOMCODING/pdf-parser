# Architecture Decision Records

## 철학

MVP는 작고 검증 가능한 Python PDF Parser를 만든다. 정확성과 명확한 실패 리포팅을 속도보다 우선하며, CLI와 라이브러리 API가 같은 핵심 파서를 공유하게 한다.

---

### ADR-001: Python CLI + 라이브러리 우선

**결정**: MVP는 웹 서버나 UI 없이 Python 패키지, CLI, 라이브러리 API로 구현한다.

**이유**: PDF 파싱의 핵심 가치는 파일 입력, 구조화 추출, 후처리 가능한 JSON 출력에 있다. CLI와 라이브러리 형태는 테스트와 자동화가 쉽고, 이후 API 서버나 UI로 확장하기 좋다.

**트레이드오프**: 비개발자용 사용성은 낮다. 파일 업로드 UI나 서버 배포 기능은 후속 범위로 둔다.

### ADR-002: PyMuPDF + pdfplumber 하이브리드 엔진

**결정**: PyMuPDF를 기본 파싱 엔진으로 사용하고, 표 추출에는 pdfplumber를 선택적으로 사용한다.

**이유**: PyMuPDF는 PDF 열기, 메타데이터, 페이지 순회, 텍스트 추출에 빠르고 범용적이다. pdfplumber는 표와 레이아웃 분석에 강하므로 표 추출 품질을 보완할 수 있다.

**트레이드오프**: 의존성이 두 개로 늘어나고 페이지별 결과를 합치는 계층이 필요하다. 엔진별 실패를 warning으로 정리하는 정책도 필요하다.

### ADR-003: 페이지 블록 중심 JSON 스키마

**결정**: JSON은 문서 메타데이터, 페이지 목록, 텍스트 블록, 표, warning, 엔진 정보를 포함하는 페이지 블록 중심 구조로 설계한다.

**이유**: 전체 텍스트만 출력하면 후처리 확장성이 낮고, 단어 단위 좌표까지 모두 보존하면 MVP 스키마와 테스트가 과해진다. 페이지 블록 중심은 실용성과 구현 범위의 균형이 좋다.

**트레이드오프**: 원본 PDF 레이아웃을 완전히 재구성하거나 단어 단위 위치 기반 처리를 하기는 어렵다. 필요한 경우 후속 버전에서 상세 좌표 모드를 추가한다.

### ADR-004: OCR은 MVP에서 제외하고 스캔 PDF는 감지만 수행

**결정**: 스캔형 PDF에 대한 OCR은 MVP에서 구현하지 않는다. 대신 텍스트가 거의 없는 PDF를 감지하고 `scanned_pdf_detected` warning을 제공한다.

**이유**: OCR은 외부 바이너리, 언어 데이터, 품질 편차, 처리 시간 문제가 크다. MVP는 텍스트 기반 PDF 파싱의 정확성을 먼저 확보해야 한다.

**트레이드오프**: 이미지 기반 문서는 실제 텍스트를 추출하지 못한다. 사용자는 warning을 보고 별도 OCR 파이프라인을 적용해야 한다.

### ADR-005: 정확성 우선과 TDD

**결정**: MVP의 우선 품질 기준은 속도보다 정확성이다. 새 기능은 테스트를 먼저 작성하고, 테스트가 통과하는 구현을 작성한다.

**이유**: PDF 파싱은 문서별 변형이 많아서 회귀가 쉽게 발생한다. 페이지 순서, 표 행/열, 실패 리포팅은 테스트로 고정해야 안정적으로 개선할 수 있다.

**트레이드오프**: 초기 구현 속도는 느려질 수 있다. 대용량 최적화와 병렬 처리는 정확성 기준이 잡힌 뒤 진행한다.

### ADR-006: 대용량 PDF는 JSONL 스트리밍 경로로 처리

**결정**: 수만 페이지 PDF 처리는 기존 `parse_pdf()`가 아니라 `parse_pdf_stream()`과 CLI `--jsonl` 모드로 제공한다. 스트리밍 record는 별도 dataclass 없이 `dict` 기반 JSONL 전송 포맷으로 둔다.

**이유**: 기존 JSON 출력은 전체 `ParsedDocument`와 전체 JSON 문자열을 메모리에 만든다. 대용량 처리의 핵심은 페이지를 하나씩 처리하고 즉시 기록하는 것이므로, record 모델을 과하게 고정하기보다 간단한 JSONL 포맷으로 메모리 사용량을 낮춘다.

**트레이드오프**: JSONL record의 타입 안정성은 도메인 모델보다 약하다. 대신 기존 공개 모델과 JSON 스키마를 유지하면서 대용량 전용 경로를 빠르게 확장할 수 있다.
